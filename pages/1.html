String Docs

1. Getting started

1.1 Declaring a string ¶

An empty string can be declared simply as follows.

string str;

So the string has been declared, what now? As of now the string does not contain any information and if you try to print it the result will be a blank output and nothing else. To make the string useful we can assign some value to it. This can be done simply by using the assignment operator '=' as follows.

string str; str = "this is a string";

Here we have assigned a traditional C-string to our string object which can be done since the string library is compatible with C-strings.

Rather than assigning a value to the string object we can also read the input from the standard input(keyboard) and store it in the string object using the stream extraction operator(>>) :

string str; std::cout << "Enter a string: "; std::cin >> str;

1.2 Printing a string ¶

Once we have data in our string object we can print it on the standard output to see its contents using the stream insertion operator(<<) :

string str; str = "this is a string"; std::cout << "The contents of the string are: "; std::cout << str;

On executing it will display:

The contents of the string are: this is a string

The stream insertion operators can be chained together as well and the above can also be written as:

string str; str = "this is a string"; std::cout << "The contents of the string are: " << str;

It will produce the same output as above on execution:

The contents of the string are: this is a string

After the string object goes out of scope any memory allocated gets freed and the programmer does not have to worry about memory management as far as the string objects are concerned.

1.3 Initializing a string

A string object can be initialized at declaration in several different ways.

1.3.1 Initializing using a default character

A string of a given size can be initialized which consists of a specified character:

string lines('-', 10); std::cout << lines;

This will generate the following output:

----------

A braced initializer can be used instead of functional initialization as follows:

string lines{'-', 10}; std::cout << lines;

This will generate the same output as above.

initialization can also be done using the '=' operator at the time of declaring the variable.

string text = "some text";

Let's take a look at an example program:

#include "arr_string.h" int main(void) { for (int i = 1; i <= 10; ++i) std::cout << string{'*', i} << '\n'; return 0; }

The code above will generate the following output:

* ** *** **** ***** ****** ******* ******** ********* **********

The program above is a simple one which contains of a for loop which is looping from 1 to 10. The loop contains a single statement which generates and prints a string of size i filled with the '*' character and then prints a newline character to get to the next line for the next iteration.

1.3.2 Initializing using a C-string literal

A string can be initialized using a C-string literal (a const char*) as follows:

string text{"this is a string"}; std::cout << text;

The output will be:

this is a string

Using this method a new string object is created and the characters of the C-string literal are copied one by one into the string.

1.3.3 Initializing using an std::string

A string can be initialized using the C++ string type std::string as well:

string str{std_string};

Here std_string is an object of type std::string.

1.3.4 Initializing using another string object

A string object can be initialized using another string object as follows:

string text{"this is a random text"}; std::cout << "text contains: " << text << '\n'; string text_copy{text}; std::cout << "text_copy contains: " << text_copy << '\n';

As we can see above we can initialize a string object from another string object. This process is also known as copy-constructing and the constructor which is used in this process is called the copy constructor. The word copy is used since the data from the source string is copied into the new string which is being created.

text contains: this is a random text text_copy contains: this is a random text

1.3.5 Initializing by moving an existing string to the new string

This may sound complicated at first but it really isn't. Whenever a new string is created some memory is allocated for it and then whatever data is to be stored in the memory is copied there. When we copy construct a string object from an existing string object we allocate memory for the new string object and then we copy the contents of the existing string to it.

When we move an existing string to a new string being created we do not allocate any new memory for the new string. The ownership of the memory allocated for the the existing string is shifted to the new string and the existing string is now empty. This is illustrated below:

#include "arr_string.h" void check_string(string& s){ if (s.is_empty()) std::cout << "The string is empty\n"; else std::cout << "The string is not empty\n"; } int main(void) { string text{"this is a string"}; std::cout << "Before moving...\n"; check_string(text); // moving the string to a new string using std::move string new_text{std::move(text)}; std::cout << "\nAfter moving...\n"; check_string(text); std::cout << "\nnew_text contains: " << new_text << '\n'; return 0; }

Generates the following output:

Before moving... The string is not empty After moving... The string is empty new_text contains: this is a string

From the above we can see that the original string becomes empty after moving it to the new string and the new string now contains what the original string contained.

1.4 Concatenating strings

Two string objects can be concatenated using the '+' operator. The '+' operator can be used to concatenate a string object and a C-string as well. Concatenating does not modify the string objects being concatenated. Both the operands to the '+' operator are left intact and a new string object containing the concatenated string is returned. If we want the contents to be modified then we have to use the '+=' operator which stores the result of the concatenation to the string object which occurs on the left hand side of the '+=' operator.

1.4.1 Concatenating two string objects

Two string objects can be concatenated using the '+' operator as follows:

#include "arr_string.h" int main(void) { string first_name{"Bruce "}, last_name{"Banner"}; std::cout << "Name: " << (first_name + last_name); return 0; }

Generates the following output:

Name: Bruce Banner

1.4.2 Concatenating a string object and a C-string

A string object can be concatenated with a C-string using the '+' operator to give a string object containing the concatenation of the two:

#include "arr_string.h" int main(void) { string username; std::cout << "Enter your username: "; std::cin >> username; std::cout << "Your login ID is: " << (username + "@gmail.com"); return 0; }

Generates the following output:

Enter your username: johnny79bravo Your login ID is: johnny79bravo@gmail.com

The order of the operands doesn't matter when concatenating i.e., it can either be string + const char* or const char* + string as shown below:

#include "arr_string.h" int main(void) { string str1{"Zig"}; const char* str2 = "Zag"; std::cout << str1 + str2 << '\n'; std::cout << str2 + str1 << '\n'; return 0; }

Generates the following output:

ZigZag ZagZig

1.4.3 Chaining the + operator

The '+' operator can be chained together to concatenate multiple strings together in any combination:

#include "arr_string.h" int main(void) { string day{"28"}, month{"8"}, year{"2023"}; string date{day + "/" + month + "/" + year}; std::cout < "Date: " < date; return 0; }

Will generate the following output:

Date: 28/8/2023

1.4.4 Compound assignment operator

The '+=' operator provides a shorthand for concatenating and storing the result in a string object. Instead of first using the '+' operator to concatenate the strings and then store the result in the string object using the copy assignment '=' operator we can use the shorthand '+=' operator to perform both the operations using a single operator as shown below:

#include "arr_string.h" int main(void) { string first_name{"Jason "}; string last_name{" Bourne"}; std::cout << "Name: " << (first_name + last_name) << '\n'; std::cout << "first_name after concatenation: " << first_name << '\n'; std::cout << "Name: " << (first_name += last_name) << '\n'; std::cout << "first_name after using '+=' operator: " << first_name << '\n'; return 0; }

Generates the following output:

Name: Jason Bourne first_name after concatenation: Jason Name: Jason Bourne first_name after using '+=' operator: Jason Bourne

1.5 Comparing strings

Strings can be compared for equality using the '==' operator. The '==' operator tests for strict equality and it can be used to compare objects of string class with C-strings as well. Other comparison operators which can be used for comparing two strings are the operators: <, >, <=, => which perform lexicographic comparisons between two string objects or a string object and a C-string.

1.5.1 Comparing two string objects for equality

Two string objects can be compared for equality using the '==' operator. The '==' operator performs a case-sensitive comparison of two strings and returns a boolean value true if the strings are exactly identical and boolean value false if the strings are not identical. The '!=' operator is the exact opposite of the '==' operator, and is used to check for inequality. Two string objects can be compared using the '==' operator as follows:

#include "arr_string.h" int main(void) { string pass{"bellissima"}, in; std::cout << "Enter password: "; std::cin >> in; if (in == pass) std::cout << "Welcome user\n"; else std::cout << "Incorrect password\n"; return 0; }

Generates the following output:

Enter password: Bellissima Incorrect password

1.5.2 Equality comparison between string objects and C-strings

The '==' operator can be used to compare string objects with C-strings as well. A string object can be compared with a C-string using the '==' operator as follows:

#include "arr_string.h" int main(void) { string in; std::cout << "Enter password: "; std::cin >> in; if (in == "bellissima") std::cout << "Welcome user\n"; else std::cout << "Incorrect password\n"; return 0; }

Generates the following output:

Enter password: Bellissima Incorrect password

Here we are using a C-string literal to compare with.

1.5.3 Case insensitive comparison

While strict comparisons provide a higher degree of accuracy, this may not always be desirable. For instance while entering names using the keyboard, different people use different combinations of upper and lowercase letters. Some people might take the time to capitalize the first letter of every word in their name while some others couldn't care less. In such cases, using case-insensitive comparisons makes more sense. The similar() method of the string class provides us with the functionality of making case-insensitive comparisons.

Let us take a look at an example:

#include "arr_string.h" int main(void) { string answer{"bjarne stroustrup"}, response; std::cout << "Who created the language C++ ? "; std::cin >> response; if (response.similar(answer)) std::cout << "You are correct\n"; else std::cout << "Incorrect answer\n"; return 0; }

Will generate the following output:

Who created the language C++ ? Bjarne Stroustrup You are correct

As we can see, the case doesn't matter when using the similar() method. Also, as was the case with the '==' operator the similar() method works for both string objects and C-strings. However since it is a class method therefore it can only be invoked using a string object and so a string object must be present on the left hand side of the '.' operator.

So the above program can also be written as:

#include "arr_string.h" int main(void) { string response; std::cout << "Who created the language C++ ? "; std::cin >> response; if (response.similar("bjarne stroustrup")) std::cout << "You are correct\n"; else std::cout << "Incorrect answer\n"; return 0; }

Will generate the following output:

Who created the language C++ ? Bjarne Stroustrup You are correct

1.5.4 Other comparison operators

The operators: <, >, <=, => can be used to compare two strings lexicographically or in other words the order in which words appear in a dictionary. These operators can be used to compare two string objects as well as for comparison between a string object and a C-string.

#include "arr_string.h" int main(void) { string a{"ask"}, b{"asks"}; std::cout << std::boolalpha; std::cout << (a < b); return 0; }

Will generate the following output:

true

1.6 Basic string manipulation

There are several methods defined in the string library to perform basic string manipulations such as changing the letter casing (lowercase or uppercase) of the string, changing the order of characters in the string, modifying the string character by character, replacing characters and so on. We will now take a look at some of the basic string manipulation methods and some examples.

1.6.1 Changing the letter casing

All the characters of a string can be converted to lowercase using the to_lower() method. The converse of this method exists which is the to_upper() method. They can be used as follows:

#include "arr_string.h" int main(void) { string text{"This is SOmE Text"}; std::cout << text.to_lower() << '\n'; std::cout << text << '\n'; text.to_upper(); std::cout << text << '\n'; return 0; }

Will generate the following output:

this is some text this is some text THIS IS SOME TEXT

We are able to use the method on the string object and use the whole expression text.to_lower() in the std::cout statement because the methods to_lower() and to_upper() return a reference to the string object.

As is obvious in the above example the string is being modified by these methods. If however there is a need to keep the string unchanged then the get_lower() and get_upper() can be used. These methods do not modify the existing string instead they return a new string object containing the desired letter casing. The same example as above will produce different output if these methods are used:

#include "arr_string.h" int main(void) { string text{"This is SOmE Text"}; std::cout << text.get_lower() << '\n'; std::cout << text << '\n'; std::cout << text.get_upper(); std::cout << text << '\n'; return 0; }

Will generate the following output:

this is some text This is SOmE Text THIS IS SOME TEXT This is SOmE Text

The original string text is not modified as seen above.

In some cases there may be a need that the first letter of the text be capitalized. The capitalize() method can be used to accomplish this task as follows:

#include <iostream> #include "arr_string.h" int main(void) { string name; std::cout << "Enter your name: "; std::cin >> name; std::cout << "You are: " << name.capitalize() << '\n'; return 0; }

Will generate the following output:

Enter your name: samudra You are: Samudra

The capitalize() method ignores any leading whitespace in the string.

1.6.2 Reversing

The reverse() method is quite basic and just reverses the string in place which means the string is modified in the process. The reversed() method on the other hand does not modify the string object but returns a new string object containing the reverse of the original string.

#include "arr_string.h" int main(void) { string text{"this text needs to be reversed"}; std::cout << text.reverse() << '\n'; std::cout << text << "\n\n"; std::cout << text.reversed() << '\n'; std::cout << text << '\n'; return 0; }

Will generate the following output:

desrever eb ot sdeen txet siht desrever eb ot sdeen txet siht this text needs to be reversed desrever eb ot sdeen txet siht

As we can see above the reverse() method modifies the string but the reversed() method doesn't.

Let us take a look at a basic program to check for palindromes using the reversed() method

#include "arr_string.h" int main(void) { string text; std::cout << "Enter some text to check: " std::cin >> text; if (text.similar(text.reversed())) std::cout << "It is a palindrome\n"; else std::cout << "It is not a palindrome\n"; return 0; }

Will generate the following output:

Enter some text to check: racecar It is a palindrome

We are using the reversed() method here since the reverse() method will first reverse the string and then the comparison would have been made to the string itself which will always result in true.

1.6.3 Collapsing whitespace

Sometimes a string may contain unwanted whitespaces in blocks. This can happen in the case of raw data entered by the user or while reading the contents from a text file. It may contain extra spaces or newline characters where not needed, carriage returns, horizontal tabs and so on. Cleaning up the string is desirable and even necessary in such cases. The string class comes with two methods collapse() and collapsed() which collapse the whitespaces in the string an replace them with single space characters(' '), thus making the text cleaner. The difference between the two methods is that collapse() modifies the string and returns a reference and collapsed() creates and returns a new string with collapsed whitespace.

#include "arr_string.h" int main(void) { string name; std::cout << "Enter name: "; std::cin >> name; std::cout << "You entered: " << name << '\n'; std::cout << "Cleaned input: " << text.collapse().title() << '\n'; return 0; }

Will generate the following output:

Enter name: thor odinson You entered: thor odinson Cleaned input: Thor Odinson

Here we have chained two method calls together. First we invoke the method collapse() to collapse the whitespace in the string and then we invoke the title() method on the result produced by the collapse() method. We are able to chain these method calls together because both of these methods return a reference to the string object. First the collapse() method is invoked and the reference is returned. Then the reference is used to invoke the title() method.

1.6.4 Replacing characters

Consider the following scenario:
Usually when using command line interfaces to create/manipulate files and directories we try to give a single word name to a file or a directory. But sometimes a one word name might not be sufficient and a descriptive name is a good thing in most cases. Then instead of separating the words in the name using spaces we use underscore characters ('_'). Converting the filenames with spaces to file names with underscores instead requires replacing the space characters with underscores.

For such applications the replace() method of the string class can be used which replaces every instance of a character in a string with any given character.

#include "arr_string.h" int main(void) { string filenames[] = { "relations and functions.pdf", "Network Programming.pdf", "journal entries.txt", "complexity analysis of algorithms.pptx", "virtual functions and runtime polymorphism in C++.pdf" }; for (string& s : filenames) s.to_lower().replace(' ', '_'); std::cout << "The modified filenames:\n"; for (string& s : filenames) std::cout << s << '\n'; return 0; }

Will generate the following output:

The modified filenames: relations_and_functions.pdf network_programming.pdf journal_entries.txt complexity_analysis_of_algorithms.pptx virtual_functions_and_runtime_polymorphism_in_c++.pdf

1.6.5 Trimming the string

Sometimes while reading input characters some trailing or leading whitespaces may appear in the string. To get rid of the unncecessary whitespaces from the ends of the string, the string class has the methods lstrip() to strip off whitespace from the left end of the string, rstrip() to strip off whitespace from the right end of the string and strip() to strip off whitespace from both the ends of the string.

#include "arr_string.h" int main(void) { string title, text; std::cout << "Enter a title for the article: "; std::cin >> title; title.strip(); std::cout << "Enter the text: "; std::cin >> text; std::cout << "\nTitle: " << title << '\n'; std::cout << "\nText: " << text; return 0; }

Will generate the following output:

Enter a title for the article: File formats Enter the text: A file format is a standard way that information is encoded for storage in a computer file. It specifies how bits are used to encode information in a digital storage medium. File formats may be either proprietary or free. Title: File formats Text: A file format is a standard way that information is encoded for storage in a computer file. It specifies how bits are used to encode information in a digital storage medium. File formats may be either proprietary or free.

The preceding and following whitespace characters are stripped from the strings.
